CS 320 Portfolio Reflection

In this course, I worked on building a contact service and writing JUnit tests to make sure the program worked correctly. I also created a summary and reflection report that explains my testing approach. These artifacts show my understanding of software testing, automation, and quality assurance. They also demonstrate how I used unit tests to find errors and confirm that my code met the requirements.

How can I ensure that my code, program, or software is functional and secure?

I can ensure my software is functional and secure by writing strong unit tests and checking my code against the requirements. In this course, I used JUnit to test different situations, including valid and invalid inputs. I tested for things like null values, incorrect lengths, and duplicate IDs. This helped confirm that my program handled errors properly. I also used code coverage tools to measure how much of my code was being tested. Making sure that most or all of the code is covered by tests helps reduce bugs and improves reliability.

How do I interpret user needs and incorporate them into a program?

I start by carefully reviewing the software requirements. I break them down into specific rules that the program must follow. For example, if a contact ID must be unique and limited to a certain length, I build those rules directly into the constructor or service methods. Then I create test cases that match those requirements. This helps make sure the program behaves the way the user expects. By connecting each test case to a requirement, I can confirm that user needs are correctly implemented.

How do I approach designing software?

I approach software design by starting simple and building step by step. First, I define the classes and their responsibilities. For example, I separated the Contact class from the ContactService class so each had a clear purpose. Then I focused on validation rules and error handling. After that, I wrote unit tests to confirm everything worked as intended. I believe designing software should include planning, clear structure, testing, and reviewing results. This process helps create clean, organized, and reliable code.
